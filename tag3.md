# Tag 3 — Git Workflow & Claude Code

**Datum:** 2026-06-05
## Was habe ich heute gelernt?

Git Workflow komplett:

- Branch erstellen und reinwechseln: git checkout -b branchname

- Dateien vormerken: git add .

- Schnappschuss erstellen: git commit -m "Nachricht"

- Branch zu GitHub schieben: git push -u origin branchname

- Pull Request auf GitHub erstellen und mergen

- Lokalen main aktualisieren: git checkout main && git pull

Claude Code Konzepte:

- Context: Das Kurzzeitgedächtnis von Claude - alles was im aktuellen Gespraech steht. Wird bei langen Gespraechen automatisch komprimiert.

- Compact: Der Befehl /compact komprimiert den Kontext manuell um Platz zu sparen ohne das Gespraech zu beenden.

- GOTCHA.md: Eine Datei im Projekt die Claude Code automatisch liest - ideal fuer Fallstricke und Besonderheiten des Projekts.

- Statusline: Zeigt Claude Code Status direkt in VS Code an - aktives Modell, Token-Verbrauch und Verbindungsstatus.
